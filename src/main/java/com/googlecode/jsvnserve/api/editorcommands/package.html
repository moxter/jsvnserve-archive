open-dir params: ( path:string parent-token:string child-token:string rev:number )
( target-rev (
rev:number target revision number
) )
Defines the target revision this delta is running for. E.g. the target revision to which an update is running.
See also: Editor.targetRevision
( open-root (
( ?rev:number ) revision number of the root (optional, could be also defined via SVN properties)
root-token:string token of the root path
) )
Opens the root directory on which the operation was invoked. All property changes as well as entries adding / deletion will be applied to this root directory. When coming back up to this root (after traversing the entire tree), the root directory must be closed with closedir.
( delete-entry (
path:string path of the entry to delete
rev:number revision number of the root (optional, could be also defined via SVN properties)
dir-token:string token of the parent path
) )
Deletes given entry depending on the "mode":
  • commit: deletes an entry within repository
  • update: deletes an entry locally (because was deleted in the repository)
  • status: informs that an entry has been deleted
( add-dir (
path:string path of the entry to add
parent-token:string token of the parent path
child-token:string token of the entry to add
( ?copy-path:string ?copy-rev:number )
) )
Adds given directory path depending on the "mode":
  • commit: adds a new directory to a repository
  • update: adds a new directory locally (because was added in the repository)
  • status: informs that a directory has been added
If copyFromPath is not null then it says that path is copied from copyFromPath located in copyFromRevision.
